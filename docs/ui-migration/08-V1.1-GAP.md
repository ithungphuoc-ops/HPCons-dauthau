## C. Chiến lược đổi token (đề xuất)

Hệ token đích theo V1.1 §B1 — trạng thái giống nhau ở cả 2 theme:

| Token mới | Giá trị | Từ token cũ | Cách xử lý |
|---|---|---|---|
| `--hp-primary` / `brand-primary` | `#096AA7` | = `brand-accent` cũ (`#0969A7`) | **Đổi giá trị `brand-primary` sang xanh dương** (~ = accent) |
| `--hp-success` | `#60BB46` | = `brand-primary` cũ | **Thêm token `success`** = giá trị xanh lá cũ |
| `--hp-warning` | `#FFA726` | `brand-warning` | Giữ nguyên |
| `--hp-danger` | `#E53935` | `brand-danger` | Giữ nguyên |
| `--hp-neutral` | `#9E9E9E` | `brand-muted` | Giữ nguyên |
| `--hp-nav-base` | `#4B4F55` | `nav-base` | Giữ nguyên |

**Cách tiếp cận an toàn nhất** (giảm rủi ro trên 152+311 chỗ):
1. Trong `index.css`: đổi `--color-brand-primary` → `#096AA7` (+ dải 50–950 sinh lại theo xanh dương); thêm `--color-brand-success` = `#60BB46` (+ dải xanh lá cũ đổi tên).
2. Vì `brand-accent` cũ ~ = primary mới → **giữ nguyên** các class `brand-accent`, hoặc trỏ `brand-accent` = `brand-primary`.
3. Rà **152 chỗ `brand-primary`**: chỗ mang nghĩa "hoàn thành / đã duyệt / đúng hạn / tiến độ đạt" → đổi class sang `brand-success`. Còn lại (nút chính, menu active, focus) tự động thành xanh dương.
4. Đổi 4 token nền dark (`#0F1720→#0F1923`, `#121C26→#131F2B`, `#18232E→#182531`, `#1E2B36→#1F2F3D`).

## E. 3 điểm cần quyết trước khi thực thi

1. **Đảo màu primary = làm lại đợt V1.0 vừa xong.** Cần xác nhận công ty **chốt V1.1** rồi mới động vào `master`; nên làm trên **nhánh riêng `ui-v1.1`** để duyệt preview trước.
2. **"Kế thừa `pkd_crm-next`"** là Next.js + shadcn/ui, còn app là **Vite + React thuần** → *không import trực tiếp được.* Đề xuất: **bám đúng spec (màu/kích thước/hành vi component)**, không đổi stack.
3. **Token đánh dấu `*` trong V1.1** (nền Light, chữ...) — tài liệu ghi rõ là *"giá trị đề xuất đội triển khai, có thể điều chỉnh khi ban hành chính thức"*. Nên hỏi lại đội thiết kế trước khi cố định màu Light Mode.
